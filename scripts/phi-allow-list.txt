# scripts/phi-allow-list.txt
#
# PHI-scan allow-list — the positive declaration that a fixture's identifiers
# are synthetic. `scripts/phi-scan.ts` reads this file; any realistic-PHI-shaped
# token NOT covered here is a hit. Byte-strict wire formats cannot carry an
# inline `# synthetic: true` header (the header segment must be the first bytes),
# so this allow-list is the synthetic-fixture declaration mechanism — the same
# approach every sibling `@cosyte/*` parser uses.
#
# Format: `<TAG> <value>` per line. `#` comments and blank lines are ignored.
#   NAME <token>          synthetic person-name word, matched case-insensitively
#   DOB <value>           synthetic date of birth (normalize as your detector does)
#   ID <value>            synthetic id matching an SSN / MRN / member-id shape
#   EMAILDOMAIN <domain>  reserved / test email domain that is allowed to appear
#
# NOTE: the floor consumes EMAILDOMAIN (SSN + email shapes). The ASTM P-record
# detector added to scripts/phi-scan.ts consumes NAME + DOB: the patient name
# tokens (P field 6) and birthdates (P field 8) of every synthetic fixture must
# be declared below, or the scanner flags them. ID entries remain reserved for a
# later phase's practice/lab-ID detector.
#
# To add a new synthetic fixture: reuse the tokens below, or append new ones here
# and commit. Adding a token is a reviewed, intentional act — never silent.
